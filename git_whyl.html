Goal

Push this local repo to GitHub with SSH authentication and keep a compact, indexed record for later study.

Steps performed

  1. Generated an ED25519 keypair locally: 
    ssh-keygen -t ed25519 -C "jfmoris110@gmail.com"
  2. Copied the public key (example): 
    ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIKjDDdp84S62hK... jfmoris110@gmail.com
    and pasted it into GitHub → Settings → SSH and GPG keys → New SSH key.
  3. Started ssh-agent and loaded the private key: 
    eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
  4. Validated SSH auth: 
    ssh -T git@github.com
# expected: Hi <account>! You've successfully authenticated, but GitHub does not provide shell access.
  5. Set remote to SSH and pushed: 
    git remote set-url origin git@github.com:jfmoris110-source/git_test.git
git push -u origin main

Notes & alternatives

  • HTTPS no longer accepts password authentication; use a PAT if you must use HTTPS.
  • The GitHub CLI (gh) can create repos and push in one step: gh repo create ... --source=. --push.
  • Procedures and commands are recorded in files in this repo (e.g. whyl_yarn.html, git_whyl.html) so they can be indexed and studied later.
